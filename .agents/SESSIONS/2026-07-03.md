### Key decisions

- **SectionTopbar is a new primitive, not a Container refactor** — `Container` has 89 importers; changing its header spacing globally was too risky. The gap-producing classes (`px-5 py-4`, header `mb-4/pb-3`, tabs `mb-6`) stay for existing pages; new contract is opt-in.
- **Trends platform-detail pages adopted alongside discovery** — they share the `SocialsNavigation` tab strip; fixing only one would make the header jump when switching tabs.
- **Reactivated dormant trends tests** — `trends/**/*.test.tsx` was outside every vitest `include` glob (never ran anywhere). Added to the pages vitest project; fixed drifted assertions to the real nav-tabs contract (`role=link` + `aria-current`, not `role=tab` + `aria-selected`) and added the missing `useRouter` mock (prefetch).
- **`OverviewTrendsPanel` defaults to `tone="card"`** so both workspace overview and brand overview harmonize; override prop kept for future shell-panel consumers.
- **Invalid thread ids behave as "no thread"** — `useAgentFullPage` sanitizes at the hook boundary rather than adding checks to each of the 3 fetch calls; route-level `replace` recovery for `/agent/undefined` URLs; store-level `upsertThread` guard stops malformed API payloads re-entering the sidebar.
