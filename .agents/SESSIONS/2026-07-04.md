## Session 1 — Design direction: monochrome hard-line + "content is the accent"

### Context

Vincent is reviewing Genfeed's design against the Vercel/Linear monochrome meta while a
website design pass runs on `claude/exciting-hodgkin-4f0020`. This session assessed the
existing token layer, made two explicit kills he ordered, and codified the strategic
design principle that distinguishes Genfeed from the dev-tool meta.

### System flow

```text
Design review question
  → survey token layer (packages/styles + packages/ui/web-tokens.css)
  → verdict: already ~80% Linear-grade monochrome (--gen-accent is literally black/white)
  → KILL 1: light-theme blue --primary/--ring → pure black (web-tokens.css + colors.ts mirror)
  → KILL 2: all glow/spotlight textures (tokens, classes, keyframes, website usages)
  → codify "Color Entry — Content Is the Accent" in DESIGN.md
  → file #1280 (content-forward studio work) as sibling to #1279 (monochrome propagation)
```

### Affected components

- **packages/ui**: `web-tokens.css` (light `--primary`/`--ring` → `0 0% 0%`), `src/core/colors.ts` (TS mirror, locked by `scripts/check-design-system.ts` drift check)
- **packages/styles**: `shadcn-theme.css` (deleted all `--shadow-glow*` tokens), `genfeed.css` (flattened `.gen-card-spotlight` to hairline card, stripped glow from `.gen-hover-lift`/`.node-card`/`.gen-card-featured`, deleted unused `.gen-hover-glow`/`.gen-hero-glow`), `keyframes.css` (deleted unused glow keyframes + animate classes)
- **apps/website**: removed `shadow-[var(--shadow-glow-*)]` from studio/publisher/workflows/integrations content pages
- **DESIGN.md**: new "Color Entry — Content Is the Accent" section + Don't bullet

### Key decisions

1. **Monochrome justified by the studio meta, not the dev-tool meta.** Genfeed's ICP is
   creators/marketers first, technical self-hosters second. Neutral chrome is the
   creative-tool rationale (Adobe/Figma/Frame.io): gray chrome keeps content color true.
   Same pixels as Vercel/Linear, different story.
2. **Color enters through exactly four doors**: user content, platform badges, semantic
   status, categorical workflow/tag palettes. Everything else grayscale. Codified in
   DESIGN.md so future PRs can be checked against it.
3. **Kill, not deprecate** (green-field): unused glow classes/keyframes deleted outright.
4. **`.gen-card-spotlight` flattened in place instead of renamed** — the class name
   survives so files deleted on the design branch (`home/_*.tsx`) don't need edits,
   avoiding modify/delete merge conflicts with `claude/exciting-hodgkin-4f0020`. Rename
   to `gen-card` once the design branch lands.
5. **Branch-deleted files left untouched on purpose** — their dangling
   `var(--shadow-glow-md)` refs resolve to no shadow now that the token is gone.
