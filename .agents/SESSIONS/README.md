# Session Notes Policy

Use one session file per day:

- `.agents/SESSIONS/YYYY-MM-DD.md`

Allowed files in this directory:

- `README.md`
- `RETENTION.md`
- `YYYY-MM-DD.md`

## Required Entry Format

Each session entry should contain:

1. Goal
2. Changes made
3. Verification commands run
4. Risks or blockers
5. Next action

## Why This Exists

Long-running model sessions degrade without compact checkpoints.
This file defines the minimum structure to preserve context quality.
