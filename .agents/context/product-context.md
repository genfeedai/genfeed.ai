# Product Context — Genfeed.ai

## Target Users

### Primary: Self-Hosted Operators
- Teams and individuals running their own AI content infrastructure
- Developers and DevOps engineers deploying Genfeed via Docker
- Organizations wanting full control over their AI content pipeline and data

### Secondary: Creators & Marketing Teams
- Content creators using AI-augmented workflows
- Marketing teams generating and distributing AI content at scale
- Agencies managing multiple brands (via enterprise `ee/` multi-tenancy)

## Core User Needs

1. **Autonomous content generation** — configure once, agents produce continuously
2. **Multi-platform distribution** — single workflow publishes to 48+ platforms
3. **Brand consistency** — maintain voice, style, and quality across all outputs
4. **Self-hosted control** — own your data, bring your own API keys, run on your infrastructure
5. **Extensibility** — customize agents, workflows, and integrations to fit your needs

## Key Features

- **Studio App**: Central workspace for brand management, agent configuration, content review
- **Agent System**: Multi-agent orchestration with specialized types (content creator, optimizer, distributor)
- **Workflow Builder**: Visual drag-and-drop workflow editor with 43+ node types
- **Content Library**: Asset management, batch generation, quality scoring
- **Integrations Hub**: 48+ platform connections (social, publishing, ads, messaging)
- **GPU Pipeline**: Image, video, and voice generation services
- **Desktop App**: Electron-based desktop client
- **Mobile App**: React Native / Expo mobile client
- **Browser & IDE Extensions**: Cross-platform extension support

## Product Model

- **Open-source (AGPL-3.0)**: Full self-hosted deployment, BYOK execution, community-driven
- **Enterprise (`ee/`)**: Multi-tenancy, SSO, advanced analytics, billing, team collaboration (commercial license)
