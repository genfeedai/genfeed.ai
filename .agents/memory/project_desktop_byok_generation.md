**Rule:** Genfeed Desktop must support local/BYOK generation without requiring
Clerk, the Genfeed API, or a local NestJS API clone.

**Current split:**
- Clerk is only required for explicit Genfeed Cloud sync/account features.
- Replicate and fal.ai BYOK image asset generation runs from Electron main,
  using the user's locally stored provider key.
- Generated BYOK assets are written into the workspace asset directory and
  registered as `local-generation`, `local-only`, `uploadPolicy: never` by
  default.
- Ollama, LM Studio, and generic OpenAI-compatible presets currently cover
  text/workflow JSON through `/chat/completions`; do not claim they generate
  image/video/audio assets until a concrete local asset adapter exists.
- Pure no-network local/offline asset generation is separate and tracked as P3
  in issue #363.

**How to apply:**
- Do not add Clerk/login gates around local/BYOK desktop generation.
- Do not silently upload generated desktop assets to Genfeed Cloud.
- Keep desktop-local execution in Electron main, not in a required local NestJS
  API process.
- Treat GitHub issue #364 as the BYOK external-provider asset adapter task and
  issue #255 as the parent desktop local generation epic.
