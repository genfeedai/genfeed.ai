**Rule:** Genfeed Desktop local/BYOK generation runs without a required account, Genfeed API session, or local NestJS API clone.

**Current split:**
- Better Auth connect is for explicit Genfeed Cloud sync/account features.
- Replicate and fal.ai BYOK image asset generation runs from Electron main, using the user's locally stored provider key.
- Generated BYOK assets are written into the workspace asset directory and registered as `local-generation`, `local-only`, `uploadPolicy: never` by default.
- Ollama, LM Studio, and generic OpenAI-compatible presets currently cover text/workflow JSON through `/chat/completions`; claim image/video/audio support only when a concrete local asset adapter exists.
- Pure no-network local/offline asset generation is tracked in issue #363.

**How to apply:**
- Keep local/BYOK desktop generation account-optional.
- Keep generated desktop assets local unless the user explicitly syncs.
- Keep desktop-local execution in Electron main rather than a required local NestJS API process.
- Treat GitHub issue #364 as the BYOK external-provider asset adapter task and issue #255 as the parent desktop local generation epic.
