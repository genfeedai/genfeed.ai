# Skills Architecture — Genfeed.ai

## Open Source Constraint

No external symlinks. No paths outside the monorepo. Clone and go.

## Directory Layout

```
genfeed.ai/
├── skills/                  # Product/content skills (used by the app)
│   ├── prompt-generator/
│   ├── workflow-creator/
│   └── ...                  # 31 content skills
├── .agents/skills/          # Dev/build skills (for building the app)
│   ├── react-patterns/
│   ├── docker-expert/
│   ├── multitenancy-guard/
│   └── ...                  # 47 dev skills
├── .claude/skills/          # GITIGNORED — auto-generated symlinks
│   └── * -> ../../.agents/skills/*
```

## Rules

1. `skills/` — product/content skills. Committed. Real files.
2. `.agents/skills/` — dev/build skills. Committed. Real files.
3. `.claude/skills/` — gitignored. Auto-generated by `scripts/setup-skills.sh`. Symlinks to `../../.agents/skills/<name>` ONLY.
4. No external symlinks. Ever.
5. `bun install` runs `postinstall` which generates `.claude/skills/` symlinks.
6. Dev skill syncs must be curated. Do not bulk-import every skill from an external repo.
7. Weekly CI validates skill integrity via `codebase-health.yml`.
