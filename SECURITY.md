# Security Policy

## Reporting a vulnerability

Do not report suspected vulnerabilities in a public issue, discussion, or pull
request. Email [support@genfeed.ai](mailto:support@genfeed.ai) with
`[SECURITY]` in the subject line.

Include:

- the affected component and deployment surface;
- a release tag, package version, or commit SHA;
- reproduction steps or a minimal proof of concept;
- the expected impact; and
- any known mitigations or workarounds.

Remove credentials, access tokens, personal data, and unrelated customer data
from the report. If sensitive test data is necessary, describe it first and wait
for a private handling method.

## Supported versions

Genfeed is under active development and does not publish an LTS or fixed support
schedule. Review releases and security advisories for disclosed fixes; older
builds may remain affected.

## Triage expectations

Reports are evaluated based on reproducibility, impact, and affected surface.
No response or remediation timeframe is guaranteed. Non-security defects belong
in the public bug report form.
