# Publishing Overview

Genfeed contains publishing integrations for multiple platforms. A connector's
presence in source does not guarantee that every account can publish every
content type: provider approval, credentials, scopes, media constraints, and
deployment mode all matter.

## Availability boundary

- Community supports manual export and locally configured integration paths.
- Cloud can provide managed publishing connectors, scheduling, and analytics.
- Hacker News and Product Hunt launch-kit support generates reviewed copy; it
  does not auto-post or automate engagement.
- Platform APIs and policies can change independently of Genfeed releases.

## Before publishing

1. Confirm that the connector is registered in your deployed release.
2. Configure the platform credential with the minimum required scopes.
3. Verify the platform account has the required API/product approval.
4. Check the connector's supported post and media types.
5. Publish a non-sensitive test item and confirm it in the provider UI.

Do not assume automatic resizing, caption optimization, hashtag generation,
optimal-time scheduling, A/B testing, analytics, or approval workflows unless
the active path visibly provides them.
