# Frequently Asked Questions

## What is Genfeed?

Genfeed is a content workflow platform with a self-hosted Community
distribution, a hosted Cloud product, and source clients for desktop, mobile,
browser, and VS Code.

## Which distribution should I use?

Use Community when you want to operate the single-tenant Docker stack and bring
your own providers. Use Cloud when you want the hosted service and managed
capabilities. The desktop client is macOS-first and local-first; mobile and
extension source does not imply a public store listing.

See [Deployment Options](/deployment).

## Which content types and models are supported?

The provider registry includes text, image, video, and voice/audio adapters.
Specific models vary with provider availability, configuration, account access,
and region. See [Available Models](/models).

## Can I collaborate with a team?

Multi-tenant organization and team-governance surfaces belong to Cloud or
commercial deployments. Community is one organization per deployment and may
enable a local Better Auth login wall.

## Is Community free to run?

Genfeed does not meter local/BYOK execution. You still pay for your own
infrastructure and any external model providers you configure. Review the
repository and package-specific licenses before redistribution.

## What does Cloud cost?

Use the [current pricing page](https://genfeed.ai/pricing). These docs do not
hard-code trial, plan, credit, seat, support, or SLA terms.

## Does self-hosting keep all data local?

Application records and local assets remain on infrastructure you operate.
Prompts and media sent to an external model or publishing provider leave the
deployment under that provider's terms. Fully local processing requires local
provider infrastructure.

## Where do I report a problem?

Use the [GitHub issue forms](https://github.com/genfeedai/genfeed.ai/issues/new/choose)
for reproducible public bugs and feature requests. No response time is
guaranteed. Do not report suspected vulnerabilities publicly; follow the
[Security Policy](https://github.com/genfeedai/genfeed.ai/security/policy).
