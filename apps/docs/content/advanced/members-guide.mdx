# Members and Roles

Genfeed's domain model includes organizations, members, and roles. Cloud and
commercial deployments can expose multi-tenant member governance; Community is
one organization per deployment.

Exact role names, permission matrices, invitation flows, and seat limits are
not treated as stable documentation here. Use the current product UI and API
schema for the deployed release, and verify authorization behavior before
automating it.

Better Auth organization records bridge authentication and sessions. Genfeed's
own organization/member/role records remain the authorization source for domain
operations.

Report access-control vulnerabilities through the
[Security Policy](https://github.com/genfeedai/genfeed.ai/security/policy), not a
public issue.
