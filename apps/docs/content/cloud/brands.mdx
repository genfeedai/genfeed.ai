# Brands

A brand is Genfeed's content context. Brand records can hold identity,
guidelines, voice/tone, assets, and provider-facing generation context.

Supported generation flows can apply configured brand context. The exact fields
and automatic behavior depend on the current UI and workflow node; these docs do
not claim that every setting is applied to every generation.

Cloud organizations can contain multiple brands. Community also supports
multiple brands inside its single organization.

Use the product's brand settings as the source of truth for the fields available
in the deployed release.
