# Genfeed Community

Community is the open-source, single-tenant Genfeed distribution. It combines
the web product, REST API, MCP server, PostgreSQL, and embedded Redis in a Docker
stack that excludes `ee/` source.

## What it is for

- visual workflow authoring;
- local or bring-your-own-key provider execution;
- text, image, video, and voice/audio provider adapters;
- brand and content-library workflows;
- REST, MCP, webhooks, and shared client packages; and
- a local workspace that does not require a Cloud account.

Model and integration availability depends on provider configuration and
provider approval. Community does not automatically replicate local records to
Cloud.

## Quick start

Use the [Community quick start](/guides/self-host-quickstart). It requires
Docker Compose, not a local Node.js/Bun toolchain.

Contributors should use [Source Installation](/core/installation) and the
[canonical contribution guide](https://github.com/genfeedai/genfeed.ai/blob/master/CONTRIBUTING.md).

## Limits and costs

Genfeed does not meter local/BYOK execution. External provider quotas and costs,
plus your own compute and storage, still apply. Prompts and media sent to an
external provider are processed under that provider's terms.

Community is one organization per deployment. Multi-tenant organizations,
managed publishing governance, managed analytics, and commercial billing are
Cloud/Enterprise product boundaries.
