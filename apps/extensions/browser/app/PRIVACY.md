# Genfeed Chrome Extension - Privacy Policy

**Last Updated:** January 27, 2025  
**Effective Date:** January 27, 2025

## Introduction

Genfeed ("we," "our," or "us") is committed to protecting your privacy. This Privacy Policy explains how we collect, use, disclose, and safeguard your information when you use the Genfeed Chrome Extension (the "Extension"). Please read this privacy policy carefully.

## Information We Collect

### 1. Authentication Information

- **OAuth Tokens**: We store authentication tokens securely in your browser's local storage (chrome.storage.local) to maintain your login session
- **User Account Data**: When you authenticate via Clerk, we receive your user ID, email address, name, and organization information
- **Session Information**: We store session data locally to keep you logged in across browser sessions

### 2. Content Data

- **Tweet Content**: When you use the Extension to generate replies or images, we temporarily process tweet content from Twitter/X to provide context for AI generation
- **Generated Content**: Content generated by the Extension (tweets, images, memes) is stored temporarily in your browser's local storage
- **Saved Tweets**: If you save tweets as inspiration, this data is stored locally and optionally synced to your Genfeed account

### 3. Usage Data

- **Feature Usage**: We may collect anonymous usage statistics about which features you use (e.g., tweet generation, image generation)
- **Error Logs**: If errors occur, we may log error information locally for debugging purposes
- **Performance Metrics**: We may collect performance data (load times, generation times) to improve the Extension

### 4. Browser Data

- **Extension Storage**: We use Chrome's local storage API to store your preferences, authentication tokens, and cached data
- **Tab Information**: We access information about open tabs to detect when you're on Twitter/X and activate Extension features
- **Cookies**: We use cookies for authentication and session management via Clerk

## How We Use Your Information

### Primary Uses

1. **Service Provision**: To provide AI-powered content generation features (tweets, replies, images, memes)
2. **Authentication**: To authenticate you with the Genfeed API and maintain your session
3. **Account Context**: To match your Twitter/X account with your Genfeed account and apply your trained style preferences
4. **Feature Improvement**: To improve our AI models and Extension functionality based on usage patterns

### Data Processing

- **AI Generation**: Tweet content and prompts are sent to our API servers for AI processing
- **Third-Party AI Services**: We use third-party AI services (OpenAI, Leonardo AI, Kling AI) to generate content. These services process your prompts but do not store your data permanently
- **Account Matching**: We match your Twitter/X username with your Genfeed account to provide personalized content generation

## Data Storage and Retention

### Local Storage

- **Authentication Tokens**: Stored in chrome.storage.local, encrypted by Chrome's security mechanisms
- **User Preferences**: Stored locally until you clear Extension data
- **Generated Content**: Stored locally for quick access, can be cleared at any time
- **Offline Queue**: Failed requests are queued locally and retried when connectivity is restored

### Server Storage

- **Generated Content**: Content generated via our API is stored on our servers for up to 30 days for your access
- **Account Data**: Your Genfeed account data (including trained styles, preferences) is stored on our servers
- **Usage Analytics**: Anonymous usage statistics may be retained for up to 2 years for analytics purposes

### Data Deletion

- You can delete locally stored data by uninstalling the Extension or clearing Extension data in Chrome settings
- To delete server-stored data, contact us at privacy@genfeed.ai
- We will delete your data within 30 days of your request, subject to legal retention requirements

## Permissions and Access

### Chrome Extension Permissions

The Extension requests the following permissions:

- **storage**: To store authentication tokens and user preferences locally
- **tabs**: To detect when you're on Twitter/X and activate Extension features
- **cookies**: For OAuth authentication and API session management
- **scripting**: To inject UI elements into Twitter/X pages
- **activeTab**: To access the current tab's content for tweet extraction
- **Host Permissions**: Access to twitter.com, x.com, and api.genfeed.ai

### How Permissions Are Used

- **storage**: Stores your login session and preferences (no external access)
- **tabs**: Detects Twitter/X pages to show Extension features (no data collection)
- **cookies**: Manages authentication cookies (only for Genfeed API)
- **scripting**: Injects "Generate Reply" and "Generate Image" buttons into Twitter/X
- **activeTab**: Extracts tweet content when you click Extension features
- **Host Permissions**: Required for Extension functionality on Twitter/X and API communication

## Your Rights and Choices

### Access and Control

- **View Your Data**: You can view locally stored data in Chrome's Extension storage viewer
- **Delete Local Data**: Clear Extension data in Chrome settings → Extensions → Genfeed → Details → Clear storage

- **Opt-Out of Analytics**: Contact privacy@genfeed.ai to opt out of usage analytics

### Data Portability

## Security

### Security Measures

- **Encryption**: All data transmitted to our API uses HTTPS/TLS encryption
- **Token Storage**: Authentication tokens are stored in Chrome's secure local storage
- **No Plaintext Storage**: We never store passwords or sensitive data in plaintext
- **Secure Authentication**: OAuth 2.0 flow with secure token refresh
- **Content Security Policy**: Extension uses strict CSP to prevent XSS attacks

### Security Best Practices

- We follow industry-standard security practices
- Regular security audits and vulnerability assessments
- Access controls and authentication for all API endpoints
- Monitoring and logging for security incidents
